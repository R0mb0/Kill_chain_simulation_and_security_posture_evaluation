Phase 1: Lab Setup and Network Topology Documentation

Project Context: Penetration Testing and Security Posture Evaluation Laboratory.
Objective: Establish a safely isolated virtual environment to simulate a Cyber Kill Chain without affecting external networks or host systems.

1. VirtualBox Internal Network Configuration

To ensure strict isolation, the laboratory relies on VirtualBox's Internal Network feature. This isolates the virtual machines from the host operating system and the internet.

  • Hypervisor: Oracle VM VirtualBox
  • Network Mode: Internal Network
  • Network Name: LabSicurezza

2. IP Addressing Plan

A static IP addressing scheme was defined for the `192.168.100.0/24` subnet to ensure stable communication between the attacking node and the vulnerable target.

Machine Role Operating System IP Address Subnet Mask
Attacker Node Kali Linux 192.168.100.10 255.255.255.0
Victim Node Metasploitable 2 192.168.100.20 255.255.255.0

3. Assigning Static IP to Metasploitable 2 (Victim)

The target machine requires a static IP assignment through its network interfaces file.

  1. Boot Metasploitable 2 and log in (default credentials: msfadmin:msfadmin).
  2. Open the network interfaces configuration file: 
    sudo nano /etc/network/interfaces
  3. Modify the eth0 configuration to apply the static IP: 
    auto eth0
iface eth0 inet static
address 192.168.100.20
netmask 255.255.255.0
  4. Save the file and restart the networking service to apply changes: 
    sudo /etc/init.d/networking restart

4. Assigning Static IP to Kali Linux (Attacker)

The attacking machine is configured via the Network Manager GUI (or CLI) to reside on the same subnet.

  1. Boot Kali Linux and navigate to the Network Manager (top-right corner) → Edit Connections.
  2. Select the active wired connection and open its settings.
  3. Under the IPv4 Settings tab, change the method from Automatic (DHCP) to Manual.
  4. Add the following configuration:
    • Address: 192.168.100.10
    • Netmask: 255.255.255.0
    • Gateway: (Leave blank to enforce isolation)
  5. Save the settings and reconnect the network interface.

5. Connectivity and Isolation Verification

To ensure the laboratory is correctly set up and securely isolated, reachability tests must be performed from the Attacker Node.

1. Verify Internal Reachability: Ping the Victim Node.

ping -c 4 192.168.100.20

Expected outcome: Successful ICMP echo replies, confirming the internal network is functional.

2. Verify External Isolation: Ping an external public server (e.g., Google DNS). 
ping -c 4 8.8.8.8
Expected outcome: "Network is unreachable" or 100% packet loss. This confirms the environment is safely contained and cannot leak traffic to the internet.
